# Production-Ready Features Roadmap

## 🎯 개발 우선순위 요약

- **Phase 1**: 핵심 기능 완성 (가장 시급)
- **Phase 2**: UX 강화
- **Phase 3**: 경쟁력 확보
- **Phase 4**: 프로덕션 준비

---

## 🔴 Phase 1: 핵심 기능 완성 (High Priority)

### 1. 카드 체크리스트 (Subtask/To-do Items)
**중요도**: ⭐⭐⭐⭐⭐

**현재 상태**:
- 부모-자식 카드만 존재
- 카드 내부 간단한 to-do 아이템 없음

**필요 기능**:
- [ ] 카드 내부에 체크박스 아이템 리스트 추가
- [ ] 체크리스트 항목 CRUD
- [ ] 진행률 표시 (예: 3/5 완료)
- [ ] 체크리스트 아이템 순서 변경
- [ ] 체크리스트 템플릿
- [ ] 완료된 항목 숨기기/보기 토글

**참고**:
- Trello, Jira에서 가장 많이 사용되는 기능
- 사용자 요구가 가장 높음

---

### 2. 카드 첨부파일 시스템
**중요도**: ⭐⭐⭐⭐⭐

**현재 상태**:
- 프로필 사진 업로드만 가능
- 카드에 파일 첨부 불가

**필요 기능**:
- [ ] 카드에 여러 파일 첨부 (이미지, PDF, 문서 등)
- [ ] 파일 업로드 API 구현
- [ ] 파일 목록 조회
- [ ] 파일 다운로드
- [ ] 파일 삭제
- [ ] 파일 미리보기 (이미지, PDF)
- [ ] 파일 크기 제한 설정
- [ ] 첨부파일 카운트 표시
- [ ] 첨부파일 검색
- [ ] 파일 버전 관리 (선택적)

**기술 고려사항**:
- 스토리지: 로컬 파일시스템 vs S3/Cloud Storage
- 파일 타입 검증
- 바이러스 스캔 (프로덕션)
- CDN 연동 (성능)

---

### 3. Due Date 리마인더 및 알림
**중요도**: ⭐⭐⭐⭐⭐

**현재 상태**:
- 마감일 설정 가능
- 알림 시스템 없음

**필요 기능**:
- [ ] 마감일 임박 자동 알림 (1일 전, 당일, 지난 후)
- [ ] 스케줄러 구현 (Spring Scheduler 또는 Quartz)
- [ ] 이메일 알림 발송
- [ ] 개인별 알림 설정 (알림 켜기/끄기)
- [ ] 알림 주기 설정 (1일 전, 3일 전 등)
- [ ] 대시보드에서 임박한 마감일 하이라이트

**기술 고려사항**:
- 배치 작업 vs 이벤트 기반
- 시간대(Timezone) 처리
- 대량 알림 성능

---

### 4. 카드 아카이브 기능
**중요도**: ⭐⭐⭐⭐

**현재 상태**:
- 보드 아카이브만 존재
- 완료된 카드 관리 기능 없음

**필요 기능**:
- [ ] 카드 아카이브 API
- [ ] 아카이브된 카드 조회
- [ ] 아카이브 복구 기능
- [ ] 아카이브된 카드 영구 삭제
- [ ] 자동 아카이브 규칙 (X일 후 자동 아카이브)
- [ ] 보드에서 아카이브 카드 뷰
- [ ] 아카이브 필터링/검색

**비즈니스 로직**:
- 아카이브 vs 삭제 차이점 명확화
- 아카이브 시 활동 로그 유지

### 18. 성능 최적화 및 확장성
**중요도**: ⭐⭐⭐⭐⭐ (프로덕션 필수)

**현재 상태**:
- H2 인메모리 DB
- 기본 구현

**필요 기능**:
- [ ] PostgreSQL/MySQL 프로덕션 DB 마이그레이션
- [ ] 데이터베이스 인덱싱 최적화
- [ ] 페이지네이션 강화 (무한 스크롤)
- [ ] Redis 캐싱 전략
  - [ ] 보드 데이터 캐싱
  - [ ] API 응답 캐싱
  - [ ] 세션 관리
- [ ] CDN 연동 (정적 파일)
- [ ] 이미지 최적화 (썸네일 생성)
- [ ] Lazy Loading
- [ ] 데이터베이스 읽기 복제본
- [ ] 쿼리 성능 모니터링
- [ ] APM (Application Performance Monitoring)

**인프라**:
- Docker/Kubernetes 배포
- CI/CD 파이프라인
- 로드 밸런싱
- Auto Scaling
