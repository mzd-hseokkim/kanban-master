# 우선순위 1 요구사항 – Modern Kanban Service

## 1. 목적
- Modern Kanban Service 1차 릴리즈에서 반드시 제공해야 하는 핵심 기능을 정의한다.
- 백엔드·프론트엔드·UX 팀이 동일한 목표와 범위를 공유하도록 기준선을 마련한다.

## 2. 범위
- **포함**: 보드·컬럼·카드 관리, 기본 협업(활동 기록, 권한), 직관적인 UX 흐름.
- **제외**: 고급 자동화, 외부 서비스 연동, 분석/리포트, 유료 기능.

## 3. 이해관계자 및 사용자
- **팀원**: 공유 보드에서 자신의 업무를 생성·추적한다.
- **프로젝트 리드**: 워크플로우를 구성하고 WIP를 관리하며 전체 상황을 모니터링한다.
- **조직 관리자**: 구성원 초대, 권한 관리, 워크스페이스 정비를 담당한다.

## 5. UX·인터랙션 요구사항
1. **빠른 진입 및 발견성**: 대시보드에서 최근 보드와 항시 노출되는 `+ 새 보드` CTA를 제공한다.
2. **일관된 컬럼 레이아웃**: 컬럼 너비와 간격은 일정해야 하며 헤더에 카드 수·WIP 상태를 노출한다.
3. **카드 가독성**: 접힌 카드에서도 제목, 담당자 아바타, 마감 배지, 라벨을 한눈에 볼 수 있어야 한다.
4. **드래그앤드롭 피드백**: 카드/컬럼 이동 시 드롭 영역 하이라이트 및 부드러운 애니메이션을 제공하고 터치 디바이스도 지원한다.
5. **빈 상태 가이드**: 빈 보드·컬럼에는 설명과 아이콘, 주요 행동 버튼(예: “첫 카드 추가”)을 표시한다.
6. **상세 패널**: 카드 상세는 패널/모달로 열리고 자동 저장, 댓글 스레드, 활동 타임라인을 포함한다.
7. **협업 표시**: 동일 카드/보드를 보고 있는 사용자의 아바타와 실시간 편집 상태를 노출한다.
8. **알림 UX**: 생성/수정/삭제 성공 메시지는 토스트 형태로, 멘션/할당은 중앙 알림 센터에서도 확인 가능해야 한다.
9. **접근성**: Glassmorphism 배경에서도 명도 대비를 확보하고 키보드 포커스 링을 모든 인터랙션 요소에 적용한다.

## 6. 비기능 요구사항
- **성능**: 카드 드래그, 컬럼 재정렬, 인라인 편집과 같은 상호작용은 150ms 이내에 반응해야 한다.
- **신뢰성**: 변경 사항은 실시간 동기화하거나 네트워크 단절 시 자동 재시도를 통해 손실을 최소화한다.
- **보안/프라이버시**: 초대된 사용자만 보드에 접근할 수 있으며 권한 검증은 클라이언트와 서버 모두에서 수행한다.
- **감사 추적**: 활동 로그는 최소 90일 이상 보관하여 회고나 컴플라이언스 요구에 대응한다.

## 7. 가정 및 제약
- 사용자 인증은 기존 IdP를 활용한다고 가정하며, 구체적 구현은 추후 확정한다.
- 1차 릴리즈는 데스크톱과 태블릿 최적화를 우선하고 모바일은 후속 우선순위에서 다룬다.
- 모든 API 엔드포인트는 `/api/v1` 프리픽스를 사용하고 REST 규약을 따른다.

## 8. 미해결 이슈
1. 아카이브된 보드·카드를 무기한 보존할지, 보존 기간을 둘지 결정 필요.
2. 우선순위 1에서 필요한 알림 채널(이메일, 푸시, 인앱)의 범위를 정의해야 한다.
3. 보드 템플릿을 조직 단위로 공유할지, 개인 단위로만 제공할지 확정이 필요하다.
